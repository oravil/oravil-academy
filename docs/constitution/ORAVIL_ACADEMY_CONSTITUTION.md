# ORAVIL ACADEMY CONSTITUTION

> **Document ID:** OA-CON-001
> **Version:** 0.1.0
> **Status:** Draft
> **Owner:** Oravil Group
> **Repository:** oravil-academy

---

# 1. Purpose

This document is the highest authority within the Oravil Academy repository.

It defines the principles, standards, and rules that govern the academy.

Every document, lesson, learning path, template, SOP, and future contribution MUST comply with this constitution.

If any document conflicts with this constitution, the constitution takes precedence.

---

# 2. Vision

Build the best Arabic practical learning platform for preparing professionals for real-world work.

The academy exists to produce capable professionals, not merely course graduates.

---

# 3. Mission

Transform knowledge into practical skills through structured learning, real projects, and measurable outcomes.

Every lesson should help learners become more independent in solving real business problems.

---

# 4. Scope

The academy includes:

- Learning Paths
- Courses
- Lessons
- Labs
- Assignments
- Knowledge Base
- SOPs
- Templates
- Case Studies
- Assessments
- Certifications

---

# 5. Non-Goals

The academy is NOT:

- a blog
- a documentation website
- a marketing website
- a video library
- a certificate factory

Every piece of content must directly support learning.

---

# 6. Core Principles

The academy follows these principles.

## Learn by Doing

Learning must always include practical application.

---

## Real Work First

Examples should be based on real business situations whenever possible.

---

## Simplicity

Prefer simple explanations over complex wording.

Complexity must never exist without purpose.

---

## Consistency

Every lesson follows the same structure.

Learners should never need to relearn navigation.

---

## Continuous Improvement

No document is considered perfect.

Every document can evolve through versioning.

---

# 7. Learning Philosophy

Every lesson should answer five questions:

1. What is it?
2. Why does it matter?
3. How does it work?
4. How is it used professionally?
5. How can the learner apply it today?

---

# 8. Repository Rules

This repository is the Single Source of Truth for the academy.

Application source code does NOT belong here.

Platform implementation lives in a separate repository.

---

# 9. Content Rules

Every lesson MUST include:

- Learning Objectives
- Explanation
- Practical Example
- Assignment
- Summary

Optional sections may be added later.

---

# 10. Contribution Rules

Every contribution must:

- improve clarity
- avoid duplicated knowledge
- follow naming conventions
- maintain consistency
- preserve learning quality

---

# 11. Definition of Done

A lesson is considered complete only if:

- objectives are defined
- explanation is complete
- example is practical
- assignment is included
- summary is written

Completion is measured by learner readiness, not document length.
